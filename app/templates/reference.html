Maturity Model Reference

Based on Continuous Delivery (Farley & Humble) and the DORA metrics framework.

Maturity Levels

LevelNameDescription
1 Regressive Manual, ad-hoc processes. Little or no automation. Tribal knowledge dominates.
2 Repeatable Some automation and documented processes. Key steps are scripted but gaps remain.
3 Consistent Standardised automation across the team. CI/CD pipeline covers build, test, and deploy.
4 Quantitatively Managed Metrics-driven. DORA metrics tracked. Decisions based on data rather than intuition.
5 Optimizing Continuous improvement and experimentation. Elite DORA performance. Self-healing systems.

What Gets Analysed

The automated assessment clones the repository and looks for the following evidence:

Version Control
  • .gitignore and README files
  • CODEOWNERS for code ownership
  • Infrastructure-as-code files (Terraform, Ansible, etc.)
  • Commit message quality and conventions
  • Branch count and discipline
Build Process
  • CI configs (GitHub Actions, Jenkins, GitLab CI, etc.)
  • Build tools (Make, Gradle, etc.)
  • Build caching and parallelisation
  • Dockerfiles for containerised builds
  • Lock files for dependency pinning
Testing
  • Test directories and test files
  • Test frameworks in dependencies
  • Tests executed in CI
  • Coverage configuration and reporting
  • E2E testing (Cypress, Playwright, etc.)
  • Performance testing tools
Deployment
  • Dockerfiles and Docker Compose
  • Kubernetes manifests, Helm charts
  • Platform configs (Heroku, Serverless, etc.)
  • CD steps in CI pipeline
  • Deployment strategies (canary, blue-green)
  • Feature flag usage
  • Release tags
Monitoring & Observability
  • Logging libraries (structlog, winston, etc.)
  • Metrics libraries (Prometheus, Datadog, etc.)
  • Tracing libraries (OpenTelemetry, Jaeger, etc.)
  • Monitoring config files
  • Alerting configuration
  • Health check endpoints in code
Security
  • SECURITY.md policy
  • Dependency scanning (Dependabot, Renovate, Snyk)
  • SAST tool configs (Semgrep, SonarQube, etc.)
  • Security scanning in CI
  • Pre-commit hooks with secret detection
  • Sensitive files excluded in .gitignore
Configuration Management
  • .env.example / environment templates
  • Config directories
  • Docker for environment parity
  • Dev containers
  • Environment-specific config files
  • GitOps tooling (ArgoCD, Flux)
Feedback Loops (DORA)
  • Release tag frequency
  • Commit frequency
  • Full CI/CD pipeline (build + test + deploy)
  • Rollback capability
AI Readiness
  • CLAUDE.md, .cursorrules, copilot-instructions
  • .claude/ directory (memory, settings, hooks)
  • AGENTS.md for multi-agent coordination
  • MCP server configuration
  • AI co-authorship percentage in git history
  • AI/ML SDKs and frameworks in dependencies
  • Agent and tool-use patterns in code
  • CodeRabbit, Aider, and other AI tool configs
